📒 Bookkeeping Reconciliation Pipeline

From raw bank PDFs to a morning digest email and a live Google Sheet tracker — end to end.

Inputs — Bank & Payroll PDFs

downloaded each month

Bank statements

Checking, savings & credit-card PDFs from BofA, Citi, Chase, BMO, Amex, US Bank, Wells Fargo, Northern Trust.

Payroll PDFs

ADP payroll reports in several layouts (1099, details, tipped, departments, labor distribution, professional).

Client config

Per-client JSON in clients/ drives detection, vendor normalization & aggregation rules — no hardcoded client names.

3

Logs — The Source of Truth

written on every confirmed run

reconciliation_log.csv

One row per reconciled account: client, account type, statement date, balances, source. The dated history the tracker reads from.

recon_log.json

Per-run status records (CLEAN / IN_PROGRESS) plus manual issue notes — feeds the email digest body.

▼
4

Unified Date Logic

get_tracker_date()

One function, two outputs

Both the email and the Google Sheet call the same get_tracker_date() in send_morning_digest.py. It picks the most recent valid date between the log and the known-good fallback, normalizes the format, and hands the identical value to both destinations — so the sheet can never drift from the email.

▼
5

Outputs

what you actually see

📧 Morning digest email

send_morning_digest.py builds an HTML email: reconciliation runs, manual notes, and a color-coded tracker grid with CC-blocking rules (what's ready, blocked, or overdue). Sent via Gmail SMTP.

📊 Google Sheet

sheets_updater.py writes to two tabs:

Tracker grid — fixed cells, latest date per account (update_sheet / --backfill).

Recon Log tab — append-only audit trail; never silently skips (append_recon_row).

🔑 Design principles

  • Config over code: client behavior lives in JSON, not hardcoded names — adding a client doesn't touch the parsers.
  • Key normalization: client-name and account-type variants are collapsed to canonical keys before the sheet lookup, fixing silent update misses.
  • Single date authority: get_tracker_date() is the one place dates are resolved, so email and sheet always agree.
  • Append-only safety net: the Recon Log tab records every run even when the fixed-cell grid has no matching cell.
